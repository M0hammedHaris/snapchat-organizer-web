
# Snapchat Organizer Web - Product Guidelines

## Product Context
This is a **marketing landing page** for the Snapchat Organizer Desktop application. The web app serves as a promotional tool to showcase features and drive desktop app downloads.

## Core Product Requirements

### Primary Purpose
- **Showcase** desktop application capabilities and features
- **Convert** visitors to desktop app downloads
- **Educate** users about Snapchat memory organization benefits
- **Build trust** through professional presentation

### Key User Journeys
1. **Discovery**: User learns about Snapchat organization challenges
2. **Education**: User understands desktop app features and benefits  
3. **Conversion**: User downloads desktop application
4. **Guidance**: User accesses setup and usage documentation

## Design System Guidelines

### Visual Identity (STRICT)
- **Theme**: Dark mode with high contrast for accessibility
- **Colors**: Use consistent accent colors throughout components
- **Typography**: Clean, readable fonts optimized for web
- **Effects**: Subtle glassmorphism and smooth transitions
- **Imagery**: High-quality mockups and screenshots of desktop app

### User Experience Principles
- **Mobile-first**: Prioritize mobile experience, enhance for desktop
- **Performance**: Fast loading times, optimized assets
- **Accessibility**: WCAG 2.1 AA compliance minimum
- **Clarity**: Clear value propositions and call-to-actions
- **Trust**: Professional design builds credibility

## Content Strategy

### Messaging Hierarchy
1. **Hero**: Clear value proposition and primary CTA
2. **Features**: Desktop app capabilities with visual proof
3. **Benefits**: User outcomes and problem solving
4. **Download**: Multiple download options and system requirements
5. **Support**: Documentation links and getting started guide

### Tone of Voice
- **Professional** yet approachable
- **Confident** in product capabilities
- **Helpful** and educational
- **Clear** and jargon-free

## Component Behavior Guidelines

### Interactive Elements
- **Buttons**: Clear hover states, loading indicators for downloads
- **Navigation**: Smooth scrolling, mobile-friendly menu
- **Forms**: Minimal friction, clear validation messages
- **Media**: Lazy loading, responsive images, video controls

### Animation Standards
- **Subtle**: Enhance UX without distraction
- **Purposeful**: Guide user attention to key actions
- **Performance**: 60fps target, respect reduced motion preferences
- **Consistent**: Unified timing and easing across components

## AI Assistant Guidelines

### When Creating Content
- Focus on **conversion optimization** - every element should drive toward download
- Emphasize **desktop app benefits** over web functionality
- Use **social proof** and **feature demonstrations** where possible
- Maintain **brand consistency** across all components

### When Implementing Features
- Prioritize **download flow optimization**
- Ensure **cross-platform compatibility** messaging
- Include **clear system requirements** and setup guidance
- Test **conversion funnel** effectiveness
